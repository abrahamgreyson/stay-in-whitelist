## Directory Purposes

**`update_whitelist/`:**
- Purpose: Main Python package containing all application logic
- Contains: Modules for IP fetching, cloud updates, logging, configuration, and cloud provider implementations
- Key files: `updater.py`, `ip_fetcher.py`, `config/config.py`

**`update_whitelist/cloud_providers/`:**
- Purpose: Cloud provider implementations following the Strategy pattern
- Contains: Abstract base class and concrete implementations for each cloud vendor
- Key files: `base_cloud_provider.py` (ABC), `huawei_cloud.py`, `tencent_cloud.py`

**`update_whitelist/config/`:**
- Purpose: Configuration loading and validation
- Contains: Pydantic model definitions and YAML loader
- Key files: `config.py` (active config with Pydantic models), `config_loader.py` (legacy, unused)

**`tests/`:**
- Purpose: Unit tests for all application modules
- Contains: One test file per source module, using pytest + pytest-mock
- Key files: All `test_*.py` files mirror the source structure

**`.github/workflows/`:**
- Purpose: CI pipeline definitions
- Contains: GitHub Actions workflow for running tests on Python 3.9-3.12

## Key File Locations

**Entry Points:**
- `main.py`: Application entry point. Starts APScheduler with 180-second interval, runs `check_and_update_ip()` on each tick.
- `update_whitelist/__init__.py`: Package init that imports version from `_version.py`.

**Configuration:**
- `config.yaml`: Runtime config with real credentials (gitignored). Loaded by `update_whitelist/config/config.py`.
- `config.example.yaml`: Template config with placeholder values. Committed to git. Used by CI via `cp config.example.yaml config.yaml`.
- `pyproject.toml`: Project metadata, dependency declarations, pytest config (`--import-mode=importlib`), setuptools_scm config.
- `update_whitelist/config/config.py`: Pydantic models (`Config`, `CloudProvider`, `Region`, `Rule`, `Allow`, `IPInfo`) and module-level `config` singleton.

**Core Logic:**
- `update_whitelist/ip_fetcher.py`: Three functions -- `get_current_ip()` calls ipinfo.io, `load_cached_ip()` reads `ip_cache.txt`, `cache_ip()` writes `ip_cache.txt`.
- `update_whitelist/updater.py`: `Updater` class -- iterates config, creates cloud clients, orchestrates delete-then-add rule updates.
- `update_whitelist/cloud_providers/base_cloud_provider.py`: ABC defining `initialize_client()`, `get_rules()`, `add_rules()`, `delete_rules()`.

**Cloud Providers:**
- `update_whitelist/cloud_providers/huawei_cloud.py`: `HuaweiCloud` -- uses Huawei VPC SDK, filters rules by "from Wulihe" description prefix.
- `update_whitelist/cloud_providers/tencent_cloud.py`: `TencentCloud` -- uses Tencent VPC SDK, filters rules by "from Wulihe" PolicyDescription prefix.

**Logging:**
- `update_whitelist/logger.py`: `get_logger()` factory returning Python logger with console + TimedRotatingFileHandler.

**Runtime Generated:**
- `ip_cache.txt`: Single-line file holding last known public IP.
- `update_whitelist.log`: Rotating log file (24-hour rotation, 7 backups).

**CI:**
- `.github/workflows/test.yml`: GitHub Actions workflow. Runs pytest with coverage on Python 3.9/3.10/3.11/3.12 matrix. Uploads to Codecov.
- `.gitlab-ci.yml`: GitLab CI pipeline with prepare + test stages. Uses custom Python 3.12 image.

## Naming Conventions

**Files:**
- Python modules: `snake_case.py` (e.g., `ip_fetcher.py`, `base_cloud_provider.py`)
- Test files: `test_<module_name>.py` (e.g., `test_updater.py`, `test_huawei_cloud.py`)
- Config files: `snake_case.yaml` or `snake_case.yml` (e.g., `config.yaml`, `test.yml`)

**Directories:**
- Python packages: `snake_case` (e.g., `cloud_providers`, `update_whitelist`)
- Test directory: Flat `tests/` directory, no nested structure mirroring

**Classes:**
- PascalCase for class names (e.g., `HuaweiCloud`, `BaseCloudProvider`, `TencentCloud`, `Updater`)
- Pydantic models also PascalCase (e.g., `CloudProvider`, `Region`, `Rule`, `Allow`, `IPInfo`)

## Where to Add New Code

**New Cloud Provider (e.g., Alibaba Cloud / Aliyun):**
1. Create new file: `update_whitelist/cloud_providers/aliyun_cloud.py`
2. Subclass `BaseCloudProvider` and implement all four abstract methods: `initialize_client()`, `get_rules()`, `add_rules()`, `delete_rules()`
3. Add the provider class import in `update_whitelist/updater.py`
4. Add a new `elif` branch in `Updater.set_client()` for the provider name (e.g., `'aliyun'`)
5. Add optional field to `Config` model in `update_whitelist/config/config.py` (field `aliyun: Optional[CloudProvider] = None` already exists)
6. Add provider config section to `config.example.yaml` and `config.yaml`
7. Create test file: `tests/test_aliyun_cloud.py`
8. Add SDK dependency to `pyproject.toml` and `requirements.txt`

**New IP Detection Service:**
- Primary code: `update_whitelist/ip_fetcher.py` -- modify or replace `get_current_ip()`
- If adding an alternate service, consider making the fetcher strategy-based similar to cloud providers

**New Configuration Fields:**
- Models: `update_whitelist/config/config.py` -- add/modify Pydantic models
- Template: Update `config.example.yaml` with new fields
- Runtime: Update `config.yaml` separately (gitignored)

**New Utility Module:**
- Place in `update_whitelist/<module_name>.py`
- Add corresponding test in `tests/test_<module_name>.py`

**New Test:**
- All tests in `tests/` directory
- Follow naming: `test_<module_under_test>.py`
- Use pytest + pytest-mock (`mocker` fixture)

## Special Directories

**`update_whitelist.egg-info/`:**
- Purpose: Setuptools build metadata
- Generated: Yes (by `pip install -e .` or `python -m build`)
- Committed: No (gitignored via `*.egg-info` pattern)

**`__pycache__/`:**
- Purpose: Python bytecode cache
- Generated: Yes (by Python interpreter)
- Committed: No (gitignored)

**`.pytest_cache/`:**
- Purpose: pytest cache for test discovery and run info
- Generated: Yes (by pytest)
- Committed: No

**`venv/`:**
- Purpose: Python virtual environment
- Generated: Yes (by `python -m venv venv`)
- Committed: No (gitignored)
