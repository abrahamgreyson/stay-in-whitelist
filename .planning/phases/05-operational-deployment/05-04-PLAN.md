# Phase 05 Plan 04: Python 3.13+ Compatibility Fix

**Objective**: Upgrade pydantic to support Python 3.13 and 3.14, ensuring the project remains compatible with current stable Python releases in 2026.

## Context

**Gap Identified**: UAT Test 1 (Cold Start Smoke Test) failed on Python 3.13.11
- **Root Cause**: pydantic 2.7.4 → pydantic-core 2.18.4 → PyO3 0.21.2 (max Python 3.12)
- **Impact**: Project cannot start on Python 3.13+, despite `requires-python = ">=3.9"` claiming support
- **Current Date**: 2026-04-08 (Python 3.14 is current stable)

**Solution**: Upgrade pydantic to 2.9+ which uses pydantic-core 2.23+ with PyO3 0.22+ (Python 3.13+ support)

## Research Summary

**pydantic version compatibility**:
- pydantic 2.7.4 → pydantic-core 2.18.4 → PyO3 0.21.2 → Python ≤3.12 ❌
- pydantic 2.9.2 → pydantic-core 2.23.4 → PyO3 0.22.0 → Python ≤3.14 ✓

**Breaking changes in pydantic 2.8+**: None that affect this project
- This project uses basic Pydantic BaseModel with simple field validators
- No deprecated APIs (model_config, field_validators are stable)
- No use of experimental features

## Execution Plan

### Task 1: Update pydantic dependency
**Goal**: Bump pydantic to 2.9+ for Python 3.13+ support

**Steps**:
1. Edit `pyproject.toml`:
   - Change `pydantic~=2.7.4` to `pydantic~=2.9.2`
   - Add `Programming Language :: Python :: 3.13` to classifiers
   - Add `Programming Language :: Python :: 3.14` to classifiers
   - Optionally add `<3.15` upper bound to `requires-python` for safety (but not required)

**Verification**:
- `grep "pydantic" pyproject.toml` shows `~=2.9.2`
- Classifiers include 3.13 and 3.14

**Commit**: `fix(deps): upgrade pydantic to 2.9.2 for Python 3.13+ support`

---

### Task 2: Install and verify dependencies
**Goal**: Ensure all dependencies resolve correctly with new pydantic

3. Verify pydantic version: Should output `2.9.2` or higher
4. Run existing test suite: `uv run pytest tests/ -v`

**Verification**:

**Commit**: No commit (verification only)
