Purpose: The current delete-before-add order creates a window where no rules exist, which locks users out if the add step fails. Define a retry decorator for cloud API calls. Per research Open Question 1, apply retry at the updater call level. The cloud provider methods already catch their own SDK exceptions and log them. For retry to work, the provider methods need to RE-RAISE network-level errors instead of swallowing them. However, since the providers currently swallow all exceptions, the retry should wrap the entire `update_security_group_rules` flow.

The simplest approach per D-12: Create a helper method `_call_with_retry` that applies tenacity to the actual cloud API method calls:

```python
# Retryable exceptions -- network level only per D-10, D-11
RETRYABLE_EXCEPTIONS = (
    requests.exceptions.ConnectionError,
    requests.exceptions.Timeout,
)

def _call_with_retry(self, fn, *args, **kwargs):
    """Execute a cloud API call with retry on transient network errors per D-09."""
    retry_decorator = retry(
        stop=stop_after_attempt(3),
        wait=wait_exponential(multiplier=1, min=1, max=10),
        retry=retry_if_exception_type(RETRYABLE_EXCEPTIONS),
        before_sleep=before_sleep_log(logger, logging.WARNING),
        reraise=True,
    )
    return retry_decorator(fn)(*args, **kwargs)
```

Then in `update_security_group_rules`, wrap the client calls:
```python
    self._call_with_retry(self.client.add_rules, sg, rules, ip)
    ...
    self._call_with_retry(self.client.delete_rules, sg, existed_rules)
```

IMPORTANT: The provider methods (huawei_cloud.add_rules, etc.) already have try/except that catches SDK exceptions. Network errors from the SDK's internal HTTP calls may be wrapped in SDK exceptions (ClientRequestException). Since providers now catch these and log them, the retry at updater level will only trigger for exceptions that escape the provider's try/except. This is correct behavior -- if the provider handles the error internally (returns None/void), no retry needed. If something unexpected escapes, retry is appropriate.

Add `import requests` at the top of updater.py (it's not currently imported there).
