<objective>
Fix cloud provider error handling to prevent unhandled exceptions and rule accumulation bugs.

Purpose: Currently huawei_cloud.add_rules() has no try/except and will crash the scheduler on API errors. Both providers' get_rules() implicitly return None on error, which causes rule accumulation (new rules added every cycle without old ones being deleted). These are the bugs that cause lockouts and garbage data.

Output: Both cloud providers return [] from get_rules() on error. huawei_cloud.add_rules() wrapped in try/except matching delete_rules pattern. All new behaviors tested.
</objective>
